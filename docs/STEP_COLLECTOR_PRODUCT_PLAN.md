## 2. למה "לא עבד פעם אחת" — ניתוח כנה

### 2.1 הבטחת מוצר מול מציאות

הארכיטקטורה נשענה על הרעיון: **אוטומציה שתחפש ותוריד מ־GrabCAD / TraceParts**.

בפועל:

- אתרים אלו משתמשים ב־**WAF / Cloudflare / anti-bot**.
- **Headless** נחסם כמעט תמיד.
- **דפדפן גלוי** עדיין דורש אימות אנושי, זמני טעינה ארוכים, והורדות שלא תמיד נופלות לתיקייה שנסרקת.
- **Playwright Chromium** בפעם הראשונה: הורדה + אנטי־וירוס = **דקות** של "תקוע", שנראה כמו תקלה.

כלומר: הליבה ששיווקנו כ־"Start וזה יוריד" **לא יציבה מספיק** ללא השקעה בתחזוקה שבועית של סלקטורים ועקיפות — וזה לא מה שהוגדר כמוצר.

### 2.2 כשלים טכניים מצטברים

- זמן אתחול Chromium / Chrome לא שקוף מספיק למשתמש (שיפרנו heartbeat ו־fallback — זה טיפול בתסמין, לא בשורש).
- מעקב אחר קבצי הורדה: **תיקיות שונות** (OneDrive, דפדפן אחר, ZIP) — דורש מודל "כל מקור אפשרי" או **רק תיקיית יעד אחת מפורשת**.
- חוויית "הצלחה" הוגדרה כ־"מצאנו hit ואז הורדנו" — במציאות רוב המשתמשים יסתיימו ב־**מסלול ידני**.

## 3. הגדרת מוצר מחדש (MVP ריאלי)

### שם עקרון

**"עוזר לארגון והשגת קבצי STEP"** — לא "בוט שמביא הכל לבד".

### מה המוצר חייב לעשות **בהצלחה 100%** (ללא תלות באתרי צד ג׳)

1. **ייבוא קבצים מקומיים** — בחירת `.step` / `.stp` / `.zip` והעתקה למבנה תיקיות אחיד (`Drone_Components_* / <component> /`).
2. **בחירת תיקיית פלט** — ברור, עם פתיחה ב־Explorer בסיום.
3. **רשימת רכיבים** — שמירה/טעינה מ־`.txt`, עריכה פשוטה.

### מה הוא עושה **כעזר** (לא כהבטחה)

4. **"פתח חיפוש"** — כפתור שפותח **את הדפדפן של המשתמש** (לא Chromium מוטמע) עם חיפוש מוכן (Google / קישורים לקטלוגים) — **בלי Playwright חובה**.
5. **תיקיית "הורדות לריצה הזו"** — נתיב אחד ברור; הוראה: "שמור את הקובץ לכאן" או גרירה לתוך החלון.

### מה מוסר או נדחה לשלב 2+

- אוטומציית login + scrape כברירת מחדל.
- תלות ב־Playwright כתנאי להפעלת האפליקציה.

## 4. ארכיטקטורה מומלצת (v2)

### אפשרות A — **מינימום שינוי (Python + CustomTkinter)**

- מסך ראשי: רשימת רכיבים + **ייבוא** + **פתח בדפדפן ברירת המחדל** ( `webbrowser.open` עם URL מוכן ).
- תיקיית `_incoming` + **שקיפות עליה** (פתח תיקייה / גרור לכאן אם נוסיף drop zone).
- Playwright — **אופציונלי**: מודול נפרד "מתקדם" שמופעל רק אם `playwright` מותקן ו־`.env` מגדיר `STEP_COLLECTOR_ENABLE_PLAYWRIGHT=1`.

### אפשרות B — **אפליקציית Web מקומית (Tauri / Electron)**

- חלון אחד עם WebView + אותה לוגיקה; קל יותר לעיצוב מדויק מ־Stitch.
- עלות: יותר תשתית.

**המלצה:** להתחיל ב־**אפשרות A** עם כיבוי Playwright כברירת מחדל — כדי שהמוצר **יעבוד תמיד**.
