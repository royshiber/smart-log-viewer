# איך מקבלים קישור שהחבר יפתח (רשת אחרת)

התוכנה רצה על **שרת בענן**. אחרי פריסה חד־פעמית תקבל כתובת כמו:

`https://שם-השירות-xxxx.onrender.com`

---

## שלב 1: קוד ב-GitHub

אם הפרויקט עדיין לא ב-GitHub:

1. צור ריפו חדש ב-[github.com](https://github.com/new)
2. דחוף את התיקייה `תוכנת לוגים` לריפו (או השתמש ב-GitHub Desktop)

---

## שלב 2: Render (חינמי, פשוט)

1. היכנס ל-[render.com](https://render.com) וצור חשבון (אפשר עם חשבון GitHub).
2. **New +** → **Web Service**.
3. חבר את הריפו של הפרויקט.
4. Render יזהה את `render.yaml` וימלא פקודות בנייה — אם לא, **ברירת המחדל של Render עובדת** אחרי עדכון הקוד האחרון:
   - **Build Command:**  
     `npm install; npm run build`  
     (או `npm ci && npm run build` — מומלץ ב־CI)
   - **Start Command:**  
     `npm run start`  
   השורש מריץ `postinstall` שמתקין תלויות ל־`client` ול־`server` אוטומטית.
5. ב-**Environment** הוסף משתנה:
   - `GEMINI_API_KEY` = המפתח מ-[Google AI Studio](https://aistudio.google.com/apikey)  
   (בלי המפתח, צ'אט Gemini לא יעבוד — שאר האפליקציה כן.)
6. לחץ **Create Web Service** והמתן לסיום הבנייה (כמה דקות בפעם הראשונה).

בסוף תראה בראש המסך **URL** — **זה הקישור לשלוח לחבר**.

---

## הערות

- שירות חינמי ב-Render **נרדם** אחרי דקות ללא שימוש; הטעינה הראשונה אחרי זה עלולה לקחת ~30–60 שניות.
- הלוגים והרכבים נשמרים ב-**IndexedDB בדפדפן של כל משתמש** — כל אחד מעלה לוגים למחשב שלו; זה לא “ענן משותף” של קבצים.

---

## בדיקה מקומית כמו בפרודקשן

```bash
npm run build --prefix client
cd server && set GEMINI_API_KEY=... && node index.js
```

פתח `http://localhost:3001` (או הפורט שמוצג בטרמינל).
